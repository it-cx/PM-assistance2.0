## 5. Tab 2：效率与能耗（efficiency）

### 5.1 页面布局

```
┌──────────────────────────────────────────────────────┐
│  stat-cards (4列)                                     │
│  [百公里能耗] [最优公司] [最差公司] [超标预警]          │
├──────────────┬───────────────────────────────────────┤
│  panel: 排名  │  panel: 单车能耗异常检测(散点图)       │
│  公司 | 车数  │  x=月里程(km), y=偏离基准(%)          │
│  |百km能耗    │  每个点=一辆车, 标签=车牌号            │
│  |vs基准|偏离 │  虚线=0%基准线                        │
├──────────────┴───────────────────────────────────────┤
│  panel: 能耗优化趋势(6月)                              │
│  [折线图 · 各公司 + 车型基准虚线]                       │
├──────────────────────────────────────────────────────┤
│  panel: 能耗健康分布                                   │
│  [柱状图 · 5档: 优秀<良好<正常<关注<偏高]              │
└──────────────────────────────────────────────────────┘
```

### 5.2 统计卡片（4 张，全按车型）

| 位置 | 标题 | 主值 | 副文字 |
|:--:|------|------|--------|
| c0 | `{车型} 百公里能耗` | `modelKwh100` kWh | `车型基准 {baseline} kWh/100km` |
| c1 | `{车型} 最优` | `best.kwh100` kWh | `{best.name} · 基准 {baseline} kWh` |
| c2 | `{车型} 最差` | `worst.kwh100` kWh | `{worst.name} · 高于基准 {worst.kwh100 - baseline} kWh` |
| c3 | 超标预警 | `overBaseline / entries.length` 组 | `⚠ 超基准 >10%` |

**数据来源：** `entries` 数组 — 按当前车型过滤，每家公司一个 `{name, idx, vehicles, mileage, energy, kwh100, baseline}` 对象。baseline 值：SD15=22, SD22C=30。

### 5.3 能耗效率排名表

**表头列定义（5 列）：**

| 列名 | 字段 | 颜色编码 |
|------|------|---------|
| 公司 | `name` + 彩色圆点 | 圆点 = `COMPANY_COLORS[idx]` |
| 车数 | `vehicles` | — |
| 百km能耗 | `kwh100` | 同颜色规则（vs 车型基准） |
| vs 基准 | `baseline` | 固定值 |
| 偏离 | `(kwh100 - baseline) / baseline × 100` | < -5% 绿/> +10% 红/> +5% 橙/±5% 正常灰 |

**排序**：按 `kwh100` 升序（默认）。`sort(function(a,b){return a.kwh100-b.kwh100})`

**交互**：点击行 → `openVehicleDetail(company, model)` → 弹窗显示该公司该车型所有车辆明细表
- 列：车辆编号 / 车型 / 月里程 / 月能耗 / 百km能耗 / vs 基准 / 评价
- 评价标签：优秀（<-10%）/ 良好（-5~-10%）/ 正常（±5%）/ 关注（+5~+15%）/ 偏高（>+15%）

### 5.4 单车能耗异常检测（effScatterChart）— 散点图

**图表定义：**

| 配置 | 值 |
|------|-----|
| 图表类型 | ECharts scatter（散点图） |
| X 轴 | 数值轴，name:'月里程 (km)'，nameLocation:'center'，nameGap:32 |
| Y 轴 | 数值轴，name:'偏离基准 (%)'，nameGap:8 |
| 每个点 | 1 辆车 = `{value:[月里程, 偏离%, 公司名, VIN, 车型, 百km能耗]}` |
| **标签** | `label: { show: true, position: 'top', formatter: 返回 VIN（如 VHC-01）}` |
| 符号大小 | `symbolSize: 14` |
| 颜色 | 统一 `modelColor`（SD15=#0EA5E9 蓝, SD22C=#F59E0B 琥珀） |
| 基准线 | `markLine: yAxis:0`，灰色虚线，标签 "0% 基准" |
| grid | `{ left:85, right:28, top:20, bottom:56 }` |

**数据生成逻辑：**
```javascript
MONTHLY[name].vehicles 中过滤 model === _currentModel
→ 每车计算: kwh100 = vehicle.energy / vehicle.mileage × 100
→ deviation = (kwh100 - baseline) / baseline × 100
→ 存入 scatterData
```

**Tooltip 格式**：`公司名 · VIN (车型)\n月里程: X km\n百km能耗: X kWh\n偏离基准: +X%`

### 5.5 能耗优化趋势图（effTrendChart）

| 配置 | 值 |
|------|-----|
| 图表类型 | 多条折线 + 1 条基准虚线 |
| X 轴 | 1月~6月 |
| Y 轴 | kWh/100km，轴名 'kWh/100km' |
| 公司线 | 每条线 = 一家公司的车型月度能耗趋势（按里程比例从趋势数据估算） |
| 基准线 | 虚线，`name: '基准 {baseline} kWh'`，颜色 `#94A3B8`，`type:'dashed'` |
| 排序 | 按当月值升序（最优在上） |
| grid | `{ left:62, right:16, top:30, bottom:24 }` |

**月度数据估算逻辑：**
```javascript
// 历史月份没有逐车型数据，使用比例估算
modelMileage = 当前月该车型总里程
allMileage  = 当前月公司总里程
ratio = modelMileage / allMileage
// 历史各月 = 公司总趋势 × ratio
historicModelMileage[i] = companyTreand.mileage[i] × ratio
historicModelEnergy[i]  = companyTreand.energy[i] × ratio
月百km能耗 = historicModelEnergy / historicModelMileage × 100
```

### 5.6 能耗健康分布图（effHealthChart）

| 配置 | 值 |
|------|-----|
| 图表类型 | 柱状图（5 柱） |
| X 轴 | 5 档标签：优秀 / 良好 / 正常 / 关注 / 偏高 |
| Y 轴 | 组数（公司-车型组合数量），`minInterval:1` |
| 颜色 | `#10B981` / `#34D399` / `#94A3B8` / `#F59E0B` / `#EF4444` |
| 数据标签 | 柱顶显示数值，`fontWeight:700` |
| grid | `{ left:48, right:16, top:28, bottom:28 }` |

**分档规则：**

| 档位 | 范围 | 含义 |
|------|------|------|
| 优秀 | 偏离 < -10% | 远低于基准 |
| 良好 | -10% ≤ 偏离 < -5% | 低于基准 |
| 正常 | -5% ≤ 偏离 ≤ +5% | 基准附近 |
| 关注 | +5% < 偏离 ≤ +15% | 高于基准 |
| 偏高 | 偏离 > +15% | 远高于基准，需干预 |

### 6.6 系统自动分析结论

3 张 `.conclusion-card`，按优先级排列：

| 卡片 | 颜色 | 内容逻辑 |
|------|------|---------|
| **crit**（红色边框） | `#DC2626` | 接管率最高的公司：`{name}：接管率 {rate} 次/百公里，为 {车型} 均值 {avgRate} 的 {倍率} 倍`。主要原因为 {该公司 TOP1 原因}（占 {占比}%），建议重点核查作业环境特征、调整路线或增加避障训练数据 |
| **warn**（黄色边框） | `#B45309` | TOP3 原因列表 + 合计占比。指出"行人/非机动车横穿"占比最高，建议集团层面统一优化避障策略和作业时段 |
| **info**（蓝色边框） | `#0369A1` | 接管率最低的公司（标杆）：建议将其作业规范作为集团标准推广 |

**数据源：**
- `rows[0]` = 接管率最高的公司（按车型过滤后的排序）
- `rows[last]` = 接管率最低的公司
- `TAKEOVER_CAUSES` top3 = 全局接管原因前三位
- `COMPANY_TAKEOVER[company]` = 该公司每种原因的接管次数（全局数据，不按车型拆分）

### 7.1 页面布局

```
┌─────────────────────────────────────────────────┐
│  stat-cards (4列)                                │

│  波动范围 + 稳定度标签                            │
└─────────────────────────────────────────────────┘
```

### 7.2 统计卡片（4 张）

| 位置 | 标题 | 主值 | 副文字 | 车型拆分 |
|:--:|------|------|--------|:--:|
| c0 | `{车型} 清扫总里程` | `modelMileage` km | `{modelVehCount} 辆车 · 日均 {modelMileage/30} km` | ✓ |
| c1 | 集团平均在线率 | `avgOnline` % | `{5} 家公司加权平均` | ✗ |
| c2 | 集团作业完成率 | `grpCompletion` % | `{done} / {total} 个任务` | ✗ |
| c3 | `{车型} 利用率达标` | `modelUtilOk / modelVehCount` 辆 | `日均里程 ≥ {std} km`（SD15=50, SD22C=55） | ✓ |

**车型拆分方式：**
- **清扫总里程**：仅统计 `model === _currentModel` 的车辆里程
- **在线率**：公司级指标，不拆分（在线状态是公司维度的属性）
- **完成率**：公司级指标，不拆分（任务不绑定车型）
- **利用率达标**：仅统计该车型车辆，达标标准使用当前车型的 `USAGE_STANDARD[model]`

### 7.3 作业完成率排名表

**表头（5 列）：** 公司 / 已完成 / 计划数 / 完成率 / `{车型} 里程`

**排序**：按 `completion` 降序

**车型里程列**：仅显示该公司该车型的里程（`modelMileage`），非公司总里程

**颜色编码：** 同通用规则（ < 70 红 / 70-80 橙 / ≥ 95 绿）

### 7.4 作业产出与质量趋势图（valueTrendChart）— 双轴

### 7.5 各公司作业稳定性热力图

**表格结构：**

| 公司 | 1月 | 2月 | 3月 | 4月 | 5月 | 6月 | 均值 | 波动 |
|------|-----|-----|-----|-----|-----|-----|------|------|
| ★龙环 | `95%`绿底 | `92%`绿底 | ... | ... | ... | ... | `91%` | `8%` 正常 |

**热力颜色映射（heatColor 函数）：**

| 完成率范围 | 背景色 | 文字色 |
|------|------|------|
| ≥ 100% | `#0F766E` 深绿 | 白 |
| 90-99% | `#047857` 绿 | 白 |
| 80-89% | `#34D399` 浅绿 | `#064E3B` 深绿 |
| 70-79% | `#FBBF24` 黄 | `#78350F` 棕 |
| 60-69% | `#EA580C` 橙 | 白 |
| < 60% | `#DC2626` 红 | 白 |

**稳定度标签（rangeTag 函数）：**

| 波动范围 | 标签 | 颜色 |
|------|------|------|
| ≤ 5% | 稳定 | 绿色 `#D1FAE5` / `#065F46` |
| 6-10% | 正常 | 灰色 `#F1F5F9` / `#64748B` |
| 11-15% | 波动 | 黄色 `#FEF3C7` / `#92400E` |
| > 15% | 不稳 | 红色 `#FEE2E2` / `#991B1B` |

**排序**：按波动范围降序 → 最不稳的排最上

**数据源**：`MONTHLY_TREND[name].completion`（6 个月完成率），波动 = `max(completion) - min(completion)`
