## PRD 审查关键决策（v1.0 → v1.1）

### 角色模型
- **无"跟车人员"角色**。现场人员 = 点位负责人，一个人负责若干收运点，车快到通知对应点位负责人前往配合
- 五个角色：管理者、调度员/操作员、点位负责人（现场）、运维人员、远程监管人员
- 车辆故障/电量危险由远程监管人员处理，不在收运 PRD 范围

### 异常兜底双策略
- **策略A**（普通点）：超时 → 自动跳过 + 通知操作员，车辆继续
- **策略B**（重点点）：超时 → 通知管理员决策；管理员超时未响应（默认10分钟）→ 自动降级为策略A
- 两层配置：重点标记→默认策略B+手动覆盖；普通→默认策略A+手动覆盖。保留灵活性

### 点位状态机
- 三态：**启用** ↔ **暂停** → **废弃**（不可逆）
- 排班模板生成任务时自动跳过暂停/废弃点位并告警
- 废弃点位在历史数据中保留，不可重新启用

### 收运确认
- 车端物理按键 + 长按 3 秒确认，物理防误触

### 通知通道分级
- 到站通知：APP 推送优先，送达失败降级为短信
- 超时告警：APP 推送 + 短信双通道
- 短信仅用于超时告警和紧急通知，需做月度成本预估（NF-05）

### 排班设计
- 模板化排班：创建一次，按周期自动生成任务
- 周期类型：每日/每周（可选周一至周日）/自定义间隔天数
- 节假日策略：跳过/正常执行/自定义班次，支持导入法定节假日日历

### 任务中动态调整
- 操作员可对执行中任务临时增删收运点、调整顺序
- 系统实时重新规划剩余路径
- 记录所有调整日志（调整人、时间、原因）

### 数据分析冷启动
- 垃圾量趋势：数据不足 7 天时显示"数据积累中（预计 X 天后可用）"
- 冷启动期用预估值作为初始参考，积累后自动切换实际趋势
- 满载率：依赖车载传感器，无传感器时显示"预估值"并标注估算依据

### 双车型共存
- 大屏同时展示清扫车（绿色图标）和收运车（蓝色图标）
- 支持按车型筛选
- 排班时两种车型不互相干扰
