## 角色与职责

**职位**：需求工程师
**所属组别**：智慧环卫组
**负责范围**：厂家端（机器人云平台）+ 运营端（环卫云平台），两端需求设计

## 上半年核心产出

### 一、厂家端（机器人云平台）

**远程驾舱功能从0到1交付（6个子功能模块）**：
- 基础远程脱困：阻塞场景下的安全远程驾驶与主动接管
- 双端协同：Android Pad + Windows桌面端，多窗口双屏自动全屏通讯验证
- 实时视频：五路摄像头同时播放
- 实时定位与座舱数据：车端数据实时刷新（每0.1s一条）
- 车辆控制与状态：监视/接管/灯光指令，车辆列表实时状态更新
- 完善阶段：远程监视、风险预警、远程对讲、驾驶预警

**平台迭代与第三方对接**：
- 功能迭代：9项
- 第三方平台对接：8家
- 累计有效运营里程：65,819.84公里
- 覆盖：20个省/直辖市

### 二、运营端（环卫云平台）

**无人清运模块设计**：
- 环服事业部数据报表看板
- 项目公司云图与数据看板
- 集团运营分析页面（5个分析Tab）
- 人车协同事件流设计

**环卫云测算工具需求分析**（计划7月完结并推广）

### 三、收运车第二车型需求分析

- 竞品分析：13家厂商三类玩家
- PRD v1.1：31项功能需求 + 5项非功能需求，6大模块
- PRD审查：18项问题闭环（4严重+8中等+6轻微）
- Issue拆分：8个垂直切片（Tracer Bullet原则）

### 四、团队建设与创新

- 基于Claude Code搭建PM专用助手体系（3个Agent + 技能链）
- 原型驱动前置验证：PRD阶段输出高保真原型
