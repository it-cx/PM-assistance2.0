# PRD：无人环卫平台 — 收运场景

**版本**: 1.1
**日期**: 2026-06-27
**状态**: Draft
**关联资料**: [竞品分析](./竞品分析-无人环卫控制平台.md)

---

## 1. 背景与目标

### 1.1 背景
无人环卫平台当前支持**道路清扫车**的监控与管理。现计划扩展第二类机器人——**无人收运车**，覆盖"收运点→中转站"的前端垃圾收运场景。竞品分析表明，行业高度聚焦清扫环节，收运环节的无人化控制平台几乎空白，是差异化切入点。

### 1.2 目标
在现有平台基础上，新增收运场景的四大核心能力：**大屏监控、收运路线规划、任务排班、数据分析**，同时复用已有的满溢检测、电量管理、车辆监控等基础能力。

---

## 2. 用户画像

| 角色 | 职责 | 关注点 |
|------|------|--------|
| **管理者** | 查看全局运营状态，做决策 | 收运完成率、异常概览、效率趋势 |
| **调度员/操作员** | 标点、排班、路线管理、异常处理 | 点位管理、排班模板、路线操作、实时告警 |
| **点位负责人（现场）** | 负责区域内若干收运点的垃圾收运配合 | 到站通知、收运确认（车端物理按键）、异常上报 |
| **运维人员** | 根据客户数据在平台标点 | 点位录入、属性配置 |
| **远程监管人员** | 处理车辆中途故障、电量危险等车辆端异常 | 车辆告警、远程干预 |

## 3. 功能需求

### 3.1 收运点管理

| ID | 功能 | 优先级 | 描述 |
|------|------|:--:|------|
| F-01 | 收运点录入 | P0 | 运维人员根据客户数据在平台地图上手动标点。支持批量导入（CSV/Excel）和单个新增 |
| F-02 | 收运点属性 | P0 | 每个点位包含：名称、地址、经纬度、类型（垃圾桶点/垃圾房）、负责人、联系电话、垃圾量预估（可后续动态校准） |
| F-03 | 异常策略配置 | P0 | 每个点位可独立配置兜底策略：**策略A**（超时离开+通知操作员）或 **策略B**（原地等待+通知上级管理员决策） |
| F-04 | 重点标记 | P1 | 点位标记"重点/普通"，默认重点点位使用策略B，普通点位使用策略A，可手动覆盖 |
| F-05 | 超时时间配置 | P1 | 每个点位可配置超时等待时间（默认值 5 分钟，可调整范围 1-15 分钟） |
| F-05.1 | 点位状态管理 | P0 | 点位支持三种状态：**启用**（正常参与排班）、**暂停**（施工/节假日等临时不可用，排班模板生成任务时自动跳过并告警）、**废弃**（永久不可用，不可被排班模板引用）。状态流转：启用↔暂停；启用→废弃（不可逆）。废弃点位在历史数据中保留，但不可重新启用 |

### 3.2 路线规划

| ID | 功能 | 优先级 | 描述 |
|------|------|:--:|------|
| F-06 | 选点排序 | P0 | 调度员从收运点库中选择本次收运的点位，拖拽调整顺序（即车辆停靠先后顺序），系统实时展示路线预览 |
| F-07 | 自动路径规划 | P0 | 根据点位顺序和地图路网，自动生成点位之间的最优行车路径。结果以地图路线形式展示 |
| F-08 | 路线保存与管理 | P0 | 规划完成的路线可保存、命名、编辑、删除。路线包含：点位列表（含顺序）、预估总里程、预估总耗时 |
| F-09 | 中转站绑定 | P0 | 每条路线需指定收运终点（中转站），满载或路线完成后车辆驶往中转站。中转站支持配置运营时段（如 6:00-22:00）和容量上限；排班时若预计到达时间不在运营时段内或容量已满，给出告警提示 |
| F-09.1 | 中转站管理 | P1 | 中转站基础信息维护：名称、位置、运营时段、容量上限、当前负载状态 |
| F-10 | 预估满载节点 | P1 | 根据各点位垃圾量预估值，预估路线上第几个点位可能触发满载，辅助调度员评估是否需要拆分路线 |

### 3.3 任务排班

| ID | 功能 | 优先级 | 描述 |
|------|------|:--:|------|
| F-11 | 排班模板创建 | P0 | 调度员创建排班模板：选择路线、指定车辆（展示载重能力与路线预估总量的对比，载重不足时给出提示但不强制阻止）、设定执行时间（时分）。支持每日多班次 |
| F-12 | 周期排班 | P0 | 按模板周期性自动生成任务。支持周期类型：每日、每周（可选周一至周日）、自定义间隔天数。支持节假日策略配置：跳过/正常执行/自定义班次，支持导入法定节假日日历 |
| F-13 | 模板管理 | P0 | 模板列表查看、启用/暂停、编辑、删除。暂停的模板不再生成新任务，不影响已生成的任务 |
| F-14 | 任务日历视图 | P1 | 以日历形式查看已生成的任务，支持按日/周/月切换，可手动调整单次任务（不修改模板） |
| F-15 | 冲突检测 | P1 | 排班时进行多维度冲突检测：同一车辆时段冲突、同一点位负责人在同一时段被两个任务通知、同一收运点在同一时段被两条路线覆盖。给出冲突提示，阻止提交或提示覆盖 |

### 3.4 收运执行与消息通知

| ID | 功能 | 优先级 | 描述 |
|------|------|:--:|------|
| F-16 | 到站通知 | P0 | 车辆行驶中实时计算距下一收运点的预计到达时间（ETA）。提前 N 分钟（可配置，默认 5 分钟）向该收运点的点位负责人推送通知（APP 推送优先，送达失败时降级为短信）。超时告警使用 APP 推送 + 短信双通道确保送达 |
| F-17 | 收运确认 | P0 | 车辆到达收运点后，点位负责人完成收运操作后，在车端按下物理按键并**长按 3 秒**确认收运完成。系统记录完成时间并触发车辆驶往下一站点。长按设计防止误触 |
| F-18 | 超时兜底-策略A | P0 | 车辆到达收运点后开始计时，超时未确认 → 自动标记该点为"跳过"，通知平台操作员，车辆按路线继续前往下一站点 |
| F-19 | 超时兜底-策略B | P0 | 车辆到达重点收运点后超时未确认 → 通知上级管理员决策（跳过 vs 继续等待）。若上级管理员超时未响应（可配置，默认 10 分钟），自动降级为策略A（跳过+通知操作员），避免车辆无限等待 |
| F-20 | 异常上报 | P1 | 点位负责人可通过 APP 上报异常（设备故障、点位不存在、垃圾桶损坏、通行受阻等），附带照片和文字说明 |
| F-21 | 任务进度实时同步 | P0 | 平台实时展示当前任务进度：已完成点数/总点数、已耗时、剩余点位、当前位置 |
| F-21.1 | 任务中动态调整 | P1 | 操作员可对执行中的任务进行动态调整：临时增删收运点、调整点位顺序。系统实时重新规划剩余路径，记录所有调整日志（调整人、时间、原因） |

### 3.5 大屏监控

| ID | 功能 | 优先级 | 描述 |
|------|------|:--:|------|
| F-22 | 地图总览 | P0 | 地图上展示所有收运点状态（待收/已完成/已跳过）、所有车辆实时位置、当前路线轨迹。支持按车型筛选（全部/清扫车/收运车），不同车型使用不同颜色图标区分。默认同时显示 |
| F-23 | 收运状态看板 | P0 | 统计卡片：今日应收点数、已完成、进行中、已跳过、待收；各路线完成百分比 |
| F-24 | 预警信息面板 | P0 | 实时滚动展示异常事件：超时跳过、等待决策、设备故障、电量告警、满溢告警 |
| F-25 | 车辆详情侧栏 | P1 | 点击车辆弹出侧栏：当前任务信息、进度、即将到达的站点、车辆状态（电量/满载率） |
| F-26 | 数据自动刷新 | P0 | 大屏数据自动刷新，延迟不超过 5 秒 |

### 3.6 数据分析

| ID | 功能 | 优先级 | 描述 |
|------|------|:--:|------|
| F-27 | 收运完成质量 | P0 | 收运完成率、跳过率、超时率——按路线/区域/时间段统计，图表+表格 |
| F-28 | 车辆运营效率 | P0 | 单趟耗时、满载率（依赖车载重量/体积传感器数据，若车辆无该传感器则显示为"预估值"并标注估算依据）、空驶里程占比、日均运行时长 |
| F-29 | 人员工作量 | P1 | 点位负责人收运配合次数、平均响应时间（通知→收运确认）、个人工作量对比 |
| F-30 | 异常分析 | P1 | 超时类型分布（策略A vs 策略B）、点位跳过频率排行、异常上报分类统计 |
| F-31 | 垃圾量趋势 | P1 | 各点位日均收运量、周/月趋势曲线，辅助排班模板优化。冷启动期（数据不足 7 天）显示"数据积累中（预计 X 天后可用）"，同时用 F-10 的垃圾量预估值作为初始参考；数据积累后自动切换为实际趋势 |

## 4. 非功能需求

| ID | 需求 | 描述 |
|------|------|------|
| NF-01 | 响应式设计 | 大屏适配 ≥1440px；管理端适配 ≥1024px；APP 端适配移动设备 |
| NF-02 | 实时性 | 车辆位置上报延迟 ≤3秒；通知推送延迟 ≤5秒；大屏刷新延迟 ≤5秒 |
| NF-03 | 可靠性 | 异常兜底逻辑在网络断开后仍可离线执行（车辆本地计时+本地决策），网络恢复后同步状态 |
| NF-04 | 可扩展性 | 平台架构需支持后续接入更多机器人类型（当前已有清扫车+收运车） |
| NF-05 | 短信成本控制 | 短信仅用于超时告警和紧急通知，不用于常规到站通知。预估月度短信量并设置成本告警阈值 |

## 5. 用户故事（核心）

- 作为调度员，我希望在地图上选择收运点并排序，系统自动生成最优路线，以便快速完成每日路线规划。
- 作为点位负责人，我希望车辆快到时收到通知，完成收运后在车端物理按键长按确认，防止误触。
- 作为管理者，我希望在大屏上看到所有收运点的完成状态和异常预警，一眼掌握全局。
- 作为调度员，我希望创建排班模板一次，系统自动按周期生成任务，避免每天重复操作。
- 作为操作员，当收运点超时未确认时，我希望收到告警，根据点位策略自动或手动决定后续行动。

## 6. 验收标准（核心路线）

### 路线：新建收运路线并排班执行

- **Given** 系统中已录入 20 个收运点，配置策略和负责人
- **When** 调度员从中选取 8 个点 → 拖拽排序 → 点击"生成路线" → 保存路线 → 创建排班模板（每日 5:00）→ 启用模板
- **Then**
  - 系统自动生成点位间最优行车路径，展示总里程和预估耗时
  - 次日 5:00 自动生成任务，分配车辆和人员
  - 车辆按路线行驶，提前 5 分钟通知每个点位的负责人
  - 大屏实时展示任务进度和车辆位置
  - 全部点位确认完成后，车辆驶往中转站

### 路线：超时兜底-策略A

- **Given** 车辆到达普通收运点（策略A，超时5分钟）
- **When** 等待 5 分钟后仍无人确认收运
- **Then** 系统自动标记该点为"跳过"，通知平台操作员，车辆继续前往下一站点

### 路线：超时兜底-策略B

- **Given** 车辆到达重点收运点（策略B，超时5分钟）
- **When** 等待 5 分钟后仍无人确认收运
- **Then** 系统通知上级管理员，车辆原地等待；管理员决定"跳过"后车辆离开，或"继续等待"后重置计时器。若管理员超时（默认 10 分钟）未响应，自动降级为策略A（跳过+通知操作员）

### 路线：每周排班模板

- **Given** 调度员创建排班模板（路线A，车辆B，每周一三五 5:00 执行）
- **When** 启用模板
- **Then** 周一/周三/周五当天 5:00 各生成一条任务；周二/周四/周六/周日不生成任务

### 路线：节假日例外

- **Given** 排班模板（每日 5:00）配置了节假日策略为"跳过"
- **When** 10月1日（国庆节）到来
- **Then** 当天不生成任务；10月8日（假期后工作日）恢复正常生成
