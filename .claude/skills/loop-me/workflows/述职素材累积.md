# 述职素材自动累积

将周报中的高价值工作提炼为述职素材条目，在半年/年终述职时自动生成初稿。

## Trigger
**事件触发**——当用户发送的内容被识别为"周报"时自动触发。识别规则：
- 用户明确说"这是本周周报"、"周报内容如下"、"本周工作"等信号词
- 内容结构匹配周报模式：分条列出本周完成的工作 + 下周计划
- 触发后自动进入入库流程，无需用户额外说"提取"

## 流程

### 入库阶段
1. 用户粘贴当周周报内容
2. AI 从中提取 1-3 条"值得进述职"的素材条目，每条一行，格式：`- YYYY-MM-DD | 项目 | 事件简述 | 价值叙事（总经理关心的角度）`
3. 用户确认（修改 / 删掉某条 / 追加）
4. 确认后的条目追加到 `述职素材.md`

### 生成阶段
1. 用户说"生成半年述职"或"生成年终述职"
2. AI 读取 `述职素材.md` 中对应时间段的条目
3. AI 按模板结构生成 markdown 初稿：
   - **上半年个人主要工作完成情况** — 从素材中挑重点工作，附带可量化结果
   - **如有未完成/未达成的工作目标** — 从素材中回溯，结合用户补充
   - **团队建设与工作创新** — 从素材中提取（如无则留空，提示用户补充）
   - **下半年个人工作计划** — 留空，由用户自行填写
   - **职业目标发展** — 留空
   - **合理化建议** — 留空
4. 模板参考：`考核评价表` 的结构（姓名/组别/职务 + 以上六个板块）
5. 输出为 markdown 文件，用户拿去修改

## 素材条目格式
```
- {日期} | {项目：云控/车联网/通用} | {事件简述} | {总经理视角的价值叙事}
```

### 示例
```
- 2026-07-03 | 云控 | 完成机器人任务调度模块重构上线 | 调度效率提升40%，支撑下半年双倍机器人接入规模，直接响应公司年度"云控平台商业化"目标
- 2026-07-05 | 车联网 | 主导车联网数据看板2.0需求设计 | 为客户运营团队提供可量化的车辆使用洞察，预计降低客户流失率15%
```

## 边界
- 每周入库时，AI 需主动过滤不值得进述职的条目（日常维护、小修小补），不要为了凑数放入
- 如果当周无可入库条目，AI 直接告知"本周无需入库"，不强行提取
- 生成述职稿时，AI 需标注每条叙述来自哪条素材，方便用户回溯验证
- 如果素材库中没有对应时间段的条目，AI 需明确告知用户，而非编造
- 非述职季（非6月/12月前后）的生成请求，AI 提醒一句"现在不是述职季，确认需要生成？"即可，不阻拦
